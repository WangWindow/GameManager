# Design System Contract

GameManager is a desktop game manager built on Tauri (Rust) + React (TypeScript). The design system document below captures the current visual implementation as a source of truth for future changes. It is not a redesign brief; it documents what is already implemented.

---

## 1. Technology Stack

| Layer | Technology | Version |
|-------|-----------|---------|
| CSS framework | Tailwind CSS | v4 |
| Component primitives | shadcn/ui | new-york style |
| Animation library | tw-animate-css | v4 compat |
| Icons | Iconify (via `@iconify/react`) | runtime |
| Toasts | Sonner | latest |
| Build | Vite | latest |

---

## 2. Token System

### 2.1 CSS Variables (source: `src/assets/app.css`)

All colors and radii are defined as CSS custom properties in `:root` (light) and `.dark` (dark) blocks. Tailwind v4 `@theme inline` maps them to utility classes.

**Base radius:** `--radius: 0.625rem` (10px). Derived radii auto-scale from it (sm/md/lg/xl/2xl/3xl/4xl).

**Color space:** OKLCH throughout.

**Light palette:**

| Token | Value |
|-------|-------|
| `--background` | `oklch(1 0 0)` white |
| `--foreground` | `oklch(0.145 0 0)` near-black |
| `--card` / `--popover` | `oklch(0.98 0 0)` |
| `--primary` | `oklch(0.205 0 0)` dark neutral |
| `--secondary` / `--muted` / `--accent` | `oklch(0.965 0 0)` light neutral |
| `--border` / `--input` | `oklch(0.90 0 0)` |
| `--ring` | `oklch(0.708 0 0)` |
| `--destructive` | `oklch(0.577 0.245 27.325)` red |

**Dark palette:**

| Token | Value |
|-------|-------|
| `--background` | `oklch(0.145 0 0)` dark neutral |
| `--foreground` | `oklch(0.985 0 0)` near-white |
| `--card` / `--popover` | `oklch(0.18 0 0)` |
| `--primary` | `oklch(0.922 0 0)` light neutral (inverted from light mode) |
| `--secondary` / `--muted` / `--accent` | `oklch(0.22 0 0)` |
| `--border` / `--input` | `oklch(1 0 0 / 15%)` translucent white |
| `--ring` | `oklch(0.556 0 0)` |

### 2.2 Dark Mode

Toggled via `.dark` class on a parent element. Tailwind `@custom-variant dark` selector: `&:is(.dark *)`. The theme mode is managed by `useThemeMode` hook and persisted.

---

## 3. Typography

Tailwind v4 defaults apply. No custom font family is set; the system font stack is used. Font sizes, weights, and line-heights follow Tailwind's utility scale. The `text-foreground` token is applied to `body` in `@layer base`.

---

## 4. Spacing & Layout

- **App window:** `#app` is 100% height, `bg-background text-foreground border border-border`, 12px border-radius, `overflow: hidden`. Background is transparent so Tauri's native window decoration can show through.
- **Component spacing:** shadcn/ui defaults. Dialog content uses `max-w-lg` (512px) for settings panels.
- **Plugin list items:** `rounded-md border px-3 py-2.5` with `space-y-1.5` vertical gaps.
- **Expanded detail panels:** `mx-4 mt-1 rounded-md border bg-muted/40 px-3 py-2 text-xs`.

---

## 5. Primitives

All interactive components come from `src/components/ui/`, generated by shadcn/ui:

- `button` -- multiple variants (default, ghost, outline, destructive)
- `dialog` -- modal with header, content, footer slots
- `card` -- surface container
- `input` -- text fields
- `select` -- dropdown
- `switch` -- toggle
- `badge` -- status indicators
- `avatar` -- game cover thumbnails
- `dropdown-menu` -- context menus
- `scroll-area` -- overflow containers
- `separator` -- horizontal dividers
- `progress` -- progress bars

Each primitive uses `border-border outline-ring/50` as its global border/outline convention.

---

## 6. Motion & Toasts

### 6.1 Sonner Toasts

Located in `src/components/ui/sonner.tsx`. Configured in `src/assets/app.css`:

- Positioned: fixed, bottom-right corner (`bottom: 1rem; right: 1rem`)
- Compact floating cards: `max-width: 320px`, `border-radius: 0.6rem`
- Shadow: `0 10px 30px rgba(15, 23, 42, 0.12)`
- Entrance animation: `translateY(4px) scale(.98)` with `opacity: 0` to full visibility over 160ms ease-out
- Exit animation: reverse via `translateY(6px) scale(.98)`
- Success toasts: green background `oklch(0.9 0.12 145)`
- Error toasts: red background `oklch(0.8 0.16 27)`, white text

### 6.2 Animation Principles

- `tw-animate-css` provides utility-based keyframe animations.
- No JavaScript-driven layout animation; all motion is CSS-only.
- Interactive states (hover, focus, active) follow shadcn/ui conventions.

---

## 7. Responsive Expectations

The app is a desktop Tauri window. Minimum design target: 800x600. No mobile responsive breakpoints are implemented. Dialogs and grids adapt to window size via intrinsic sizing (flex-wrap, max-width constraints) rather than explicit breakpoints.

---

## 8. Accessibility & Focus

- Border/ring outline convention: `outline-ring/50` applied globally to `* { @apply border-border outline-ring/50 }`
- Focus-visible rings use the `--ring` token color
- Keyboard navigation follows shadcn/ui dialog and dropdown defaults
- Switch toggles are disabled when the underlying plugin is invalid

---

## 9. Accepted Design Debt

- Icons are sometimes loaded as raw Iconify strings (e.g., `ri:plugin-line`) inline in JSX rather than through a centralized icon component or sprite sheet. This is acceptable for the current scale.
- No design tokens are exported for Figma or non-code tooling.
- Dark mode toggle has no smooth transition animation; it swaps instantly via class change.
- The game card cover images do not enforce a consistent aspect ratio; they stretch to fill the card area.
- Toast styles use manual CSS in app.css instead of Sonner's theming API.
- No loading skeleton is used for async plugin detail expansion; a plain spinner and "Loading..." text are rendered instead.
